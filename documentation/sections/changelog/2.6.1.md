*   The `evelentscript` package itself now supports named exports when used by ES modules in Node.js; or in other words, `import { compile } from 'evelentscript'` now works, rather than only `import EvelentScript from 'evelentscript'`.
*   Bugfix for a stack overflow error when compiling large files in non-bare mode.
